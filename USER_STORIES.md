# Screen Recorder — User Stories

## 1. Запись экрана

### 1.1 Начать запись
**Как** пользователь, **я хочу** нажать одну кнопку (Record), **чтобы** начать запись экрана с аудио с микрофона.

Принятие:
- Кнопка Record запускает запись в формате MP4 (H.264 + AAC)
- При старте кнопка меняется на Stop
- Видео сохраняется в выбранную папку (по умолчанию ~/Movies на macOS)
- Имя файла: `recording_YYYY-MM-DD_HH-MM-SS.mp4`

### 1.2 Остановить запись
**Как** пользователь, **я хочу** нажать Stop, **чтобы** завершить запись и получить готовый файл.

Принятие:
- Запись останавливается, файл финализируется
- Показывается имя сохранённого файла
- Запись автоматически попадает в историю
- После записи автоматически запускается cleanup (если включён)

### 1.3 Видеть статус во время записи
**Как** пользователь, **я хочу** видеть таймер и размер файла в реальном времени, **чтобы** контролировать длительность и объём записи.

Принятие:
- Мигающий индикатор REC
- Таймер в формате MM:SS
- Размер файла обновляется каждую секунду

### 1.4 Видеть текущий пресет перед записью
**Как** пользователь, **я хочу** видеть название активного пресета на главном экране, **чтобы** знать, с каким качеством будет запись.

Принятие:
- Тег с именем пресета рядом со статусом "Ready"
- При отсутствии пресета — нет тега

### 1.5 Горячая клавиша для записи
**Как** пользователь, **я хочу** запускать/останавливать запись горячей клавишей (⌘+Shift+R), **чтобы** не переключаться в окно приложения.

Принятие:
- Глобальный шорткат ⌘+Shift+R (macOS) / Ctrl+Shift+R (Win/Linux)
- Переключает запись (start/stop)
- Работает когда приложение свёрнуто

### 1.6 Управление из трея
**Как** пользователь, **я хочу** управлять записью из системного трея, **чтобы** быстро начать/остановить запись без открытия окна.

Принятие:
- Иконка в трее
- Контекстное меню: Start Recording / Stop Recording / Quit
- Команды тригерят те же действия, что и кнопки в UI

## 2. Разрешения (macOS)

### 2.1 Проверка разрешения на запись экрана
**Как** пользователь, **я хочу** видеть предупреждение, если у приложения нет разрешения на запись экрана, **чтобы** знать, почему запись не работает.

Принятие:
- При старте приложения проверяется разрешение (CGPreflightScreenCaptureAccess + test capture)
- Если нет — показывается жёлтый баннер с инструкцией
- Подсказка: "Toggle OFF → ON for Screen Recorder, then restart app"

### 2.2 Открыть настройки безопасности
**Как** пользователь, **я хочу** нажать кнопку "Open Settings" в баннере, **чтобы** быстро перейти к настройкам macOS для выдачи разрешения.

Принятие:
- Открывает System Settings → Privacy & Security → Screen Recording
- Работает через URL-схему `x-apple.systempreferences:`

## 3. Настройки

### 3.1 Выбрать папку для сохранения
**Как** пользователь, **я хочу** выбрать папку, куда сохраняются записи, **чтобы** хранить файлы в удобном месте.

Принятие:
- Кнопка Change открывает нативный диалог выбора папки
- Отображается текущий путь (сокращённо, с ~)
- Кнопка Reset возвращает к значению по умолчанию (~/Movies)
- Настройка сохраняется в settings.json

### 3.2 Выбрать пресет качества
**Как** пользователь, **я хочу** выбрать пресет качества из выпадающего списка, **чтобы** управлять соотношением качество/размер.

Принятие:
- Выпадающий список всех доступных пресетов
- Каждый пункт: Название + описание (например "Voice-first (~95-130 MB/h)")
- Выбор сохраняется и применяется к следующей записи

### 3.3 Выбрать микрофон
**Как** пользователь, **я хочу** выбрать устройство аудиозахвата, **чтобы** записывать звук с нужного микрофона.

Принятие:
- Выпадающий список обнаруженных аудиоустройств
- По умолчанию: первое найденное устройство
- Если микрофон недоступен — запись продолжается без аудио (video only)

### 3.4 Настройки сохраняются между сессиями
**Как** пользователь, **я хочу** чтобы мои настройки сохранялись, **чтобы** не перенастраивать приложение каждый раз.

Принятие:
- Все настройки персистятся в ~/.config/screen-recorder/settings.json
- Загружаются при старте приложения
- Обратная совместимость: legacy поле `quality` мигрируется в `selected_preset_id`

### 3.5 Навигация в настройки
**Как** пользователь, **я хочу** попасть в настройки через кнопку "Settings" на главном экране, **чтобы** быстро менять параметры.

Принятие:
- Кнопка Settings в навигационной панели
- Недоступна во время записи
- Кнопка "← Back" возвращает на главный экран

## 4. Пресеты качества

### 4.1 Просмотр списка пресетов
**Как** пользователь, **я хочу** видеть все доступные пресеты с их параметрами, **чтобы** понять различия и выбрать подходящий.

Принятие:
- Список с параметрами: высота, FPS, CRF, x264 preset, каналы аудио, описание
- Теги: "system" (системные), "active" (текущий выбранный)
- Кнопки действий: Select / Edit / Dup / Del

### 4.2 Создать пользовательский пресет
**Как** пользователь, **я хочу** создать свой пресет с нужными параметрами, **чтобы** настроить качество под конкретную задачу.

Принятие:
- Кнопка "+ New" открывает редактор с дефолтными значениями
- Поля: ID, Name, Description, Height, FPS, CRF (0-51), x264 preset, Audio bitrate, Channels (mono/stereo)
- Валидация: fps > 0, height > 0, crf 0-51, channels 1 или 2, валидное имя x264 preset
- ID не может быть пустым, не может содержать пробелы
- При ошибке — сообщение об ошибке

### 4.3 Редактировать пресет
**Как** пользователь, **я хочу** изменить параметры существующего пресета, **чтобы** подстроить качество.

Принятие:
- Кнопка Edit открывает редактор с текущими значениями
- ID нельзя изменить при редактировании
- Системные пресеты тоже можно редактировать
- Поле `updated_at` обновляется при сохранении

### 4.4 Дублировать пресет
**Как** пользователь, **я хочу** клонировать существующий пресет, **чтобы** создать вариацию без ввода всех параметров заново.

Принятие:
- Кнопка Dup создаёт копию с суффиксом "-copy" в ID и "(Copy)" в названии
- Копия — пользовательский пресет (is_system = false)
- Копия сразу появляется в списке

### 4.5 Удалить пользовательский пресет
**Как** пользователь, **я хочу** удалить ненужный пресет, **чтобы** не захламлять список.

Принятие:
- Кнопка Del доступна только для пользовательских пресетов (не system)
- Если удаляется активный пресет — автоматически выбирается первый доступный
- Системные пресеты нельзя удалить

### 4.6 Четыре системных пресета по умолчанию
**Как** пользователь, **я хочу** иметь готовые пресеты для типичных сценариев, **чтобы** не настраивать всё вручную.

Принятие:
- **Voice-first**: 540p, 5 fps, CRF 34, mono, ~95-130 MB/h — для аудио-записей
- **Daily Lite**: 720p, 8 fps, CRF 30, mono, ~200 MB/h — повседневный
- **Meeting**: 900p, 15 fps, CRF 26, stereo, ~450 MB/h — для митингов
- **Presentation**: 1080p, 24 fps, CRF 22, stereo, ~900 MB/h — для презентаций
- Все помечены как is_system = true
- Ширина вычисляется из aspect ratio экрана (не фиксированная)

### 4.7 Пресеты хранятся в JSON-файле
**Как** продвинутый пользователь, **я хочу** редактировать пресеты напрямую в JSON, **чтобы** иметь полный контроль.

Принятие:
- Файл ~/.config/screen-recorder/presets.json
- Если файл отсутствует или повреждён — генерируются дефолтные пресеты
- При первом запуске с несуществующим selected_preset_id — фолбэк на первый доступный

## 5. История записей

### 5.1 Просмотр истории
**Как** пользователь, **я хочу** видеть список всех записей, **чтобы** быстро найти и открыть нужную.

Принятие:
- Список отсортирован от новых к старым (до 200 записей)
- Каждая запись: имя файла, дата, размер, длительность, название пресета
- Тег "missing" для файлов, которые были удалены вне приложения
- Количество записей в заголовке: "History (N)"

### 5.2 Открыть запись
**Как** пользователь, **я хочу** открыть файл записи, **чтобы** просмотреть его в плеере.

Принятие:
- Кнопка Open открывает файл в системном плеере
- Доступна только для существующих файлов

### 5.3 Показать в папке
**Как** пользователь, **я хочу** показать файл в файловом менеджере, **чтобы** быстро найти его на диске.

Принятие:
- Кнопка Reveal открывает Finder/Explorer с выделенным файлом
- Доступна только для существующих файлов

### 5.4 Удалить запись
**Как** пользователь, **я хочу** удалить файл записи, **чтобы** освободить место.

Принятие:
- Кнопка Delete удаляет файл с диска И убирает из истории
- Доступна только для существующих файлов
- Удаление безвозвратное

### 5.5 Убрать из истории
**Как** пользователь, **я хочу** убрать запись из истории (без удаления файла), **чтобы** навести порядок.

Принятие:
- Кнопка Remove убирает запись из списка, файл не трогает
- Доступна для записей с тегом "missing"

### 5.6 Пересканировать историю
**Как** пользователь, **я хочу** обновить статусы файлов, **чтобы** увидеть актуальное состояние.

Принятие:
- Кнопка Rescan проверяет все записи на существование файлов
- Обновляет статус "exists"/"missing" и размеры файлов
- Обновлённый список отображается сразу

### 5.7 История обновляется после каждой записи
**Как** пользователь, **я хочу** чтобы новая запись автоматически появилась в истории, **чтобы** не обновлять список вручную.

Принятие:
- После остановки записи — запись добавляется в начало истории
- Метаданные: размер файла, длительность, дата, использованный пресет
- История лимитирована 500 записями (показывается 200)

## 6. Политика хранения (Storage Policy)

### 6.1 Включить/выключить автоочистку
**Как** пользователь, **я хочу** включить автоматическую очистку старых записей, **чтобы** не заполнять диск.

Принятие:
- Чекбокс "Enable auto-cleanup"
- По умолчанию выключен
- При включении — показываются дополнительные настройки

### 6.2 Настроить срок хранения
**Как** пользователь, **я хочу** задать, сколько дней хранить записи, **чтобы** контролировать объём.

Принятие:
- Выбор из: 7 / 30 / 90 / 180 / 365 дней
- По умолчанию: 30 дней
- Файлы старше этого срока удаляются при cleanup

### 6.3 Удалять большие файлы раньше
**Как** пользователь, **я хочу** автоматически удалять крупные файлы уже через 1 день, **чтобы** экономить место.

Принятие:
- Чекбокс "Also delete large files older than 1 day"
- Настраиваемый порог: Large file threshold (MB), по умолчанию 500 MB
- Минимум 50 MB, шаг 50 MB

### 6.4 Лимит удалений за один прогон
**Как** пользователь, **я хочу** ограничить количество файлов, удаляемых за один запуск, **чтобы** избежать случайного удаления всего.

Принятие:
- Поле "Max files to delete per run"
- По умолчанию: 20
- Диапазон: 1-100

### 6.5 Запустить очистку вручную
**Как** пользователь, **я хочу** запустить cleanup вручную, **чтобы** освободить место прямо сейчас.

Принятие:
- Кнопка "Run cleanup now"
- Показывает результат: "Cleaned up N file(s)" или "Nothing to clean up"
- Сообщение исчезает через 3 секунды

### 6.6 Автоочистка при старте и после записи
**Как** пользователь, **я хочу** чтобы cleanup запускался автоматически, **чтобы** не помнить о ручной очистке.

Принятие:
- Cleanup запускается при старте приложения
- Cleanup запускается после каждой остановки записи
- Только если cleanup_enabled = true

### 6.7 Безопасность cleanup
**Как** пользователь, **я хочу** чтобы cleanup удалял только мои записи, **чтобы** случайно не потерять другие файлы.

Принятие:
- Удаляются только файлы, находящиеся внутри папки записей (recordings_dir)
- Файлы за пределами папки записей — игнорируются
- Удаляются только файлы из истории (не произвольные файлы в папке)

## 7. Ретина / HiDPI

### 7.1 Корректная запись на Retina-дисплеях
**Как** пользователь с Retina-дисплеем, **я хочу** чтобы записи имели правильное разрешение, **чтобы** видео не было в 2x или размытым.

Принятие:
- Экран захватывается в физическом разрешении (e.g. 2992x1934)
- Энкодер масштабирует через swscale до логического разрешения (e.g. 1496x967)
- Выходной файл имеет корректные размеры на основе target_height пресета
- Ширина рассчитывается с учётом aspect ratio экрана

---

## 8. Аудио

### 8.1 Запись аудио с микрофона
**Как** пользователь, **я хочу** чтобы видео содержало звук с микрофона, **чтобы** записать голосовой комментарий.

Принятие:
- Аудио записывается через cpal (системный API)
- Кодируется в AAC
- Синхронизировано с видео (A/V sync через frame-count PTS)

### 8.2 Запись без аудио при ошибке
**Как** пользователь, **я хочу** чтобы запись продолжалась даже если микрофон недоступен, **чтобы** не потерять видео.

Принятие:
- Если аудиоустройство недоступно — записывается только видео
- Предупреждение в логах: "Audio not available: ... Video only."
- Приложение не крашится

## 9. Кросс-платформенность

### 9.1 macOS: полная поддержка
- Screen Recording permission management
- Retina handling
- ~/Movies как дефолтная папка
- System tray

- ~/Videos как дефолтная папка
- System tray

## 10. Навигация и UX

### 10.1 Шесть экранов приложения
**Как** пользователь, **я хочу** переключаться между экранами, **чтобы** получить доступ ко всем функциям.

Экраны:
1. **Main** — запись, статус, навигация
2. **Settings** — папка, пресет, микрофон
3. **Storage** — политика хранения (из Settings)
4. **History** — список записей
5. **Presets** — список пресетов
6. **Preset Edit** — редактор одного пресета (из Presets)

### 10.2 Навигация заблокирована во время записи
**Как** пользователь, **я хочу** чтобы во время записи были недоступны настройки и пресеты, **чтобы** случайно не изменить параметры.

Принятие:
- Кнопки Settings и Presets disabled во время записи
- History доступна всегда

### 10.3 Компактный минималистичный UI
**Как** пользователь, **я хочу** маленькое аккуратное окно, **чтобы** приложение не мешало работе.

Принятие:
- Тёмная тема (#111827 фон)
- Компактные элементы, маленький шрифт
- Круглая кнопка записи (90×90px)
- Панели с прокруткой для длинного контента

## 11. Сборка и деплой

### 11.1 rebuild.sh — полный цикл
**Как** разработчик, **я хочу** одной командой пересобрать и установить приложение, **чтобы** ускорить итерации.

Шаги:
1. Убить все процессы
2. Deep clean: tccutil reset, удалить все копии .app, кэши, savedState, preferences
3. `npm run tauri build`
4. Скопировать в ~/Applications/
5. Запустить
6. Показать инструкцию по разрешениям

### 11.2 Упаковка в .app и .dmg
**Как** пользователь, **я хочу** получить .app и .dmg, **чтобы** установить приложение стандартным для macOS способом.

Принятие:
- Tauri генерирует Screen Recorder.app
- Tauri генерирует Screen Recorder_0.1.0_aarch64.dmg
